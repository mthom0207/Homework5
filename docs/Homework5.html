3. What are two advantages of using purrr functions instead of the BaseR apply family?

  • Main advantage of purrr functions in comparison to the BaseR apple family is more consistency/cleaner way to apply functions to objects and has lots of additional helper functions.

  • Purrr functions also give a shorthand way to make anonymous functions

4. What is a side-effect function?

Side-effect functions are functions that affect the state of the world in any way apart from the value they return.

5. Why can you name a variable sd in a function and not cause any issues with the sd function?

When R evaluates the code inside your function, it first looks in the function’s local environment for the name sd. If it finds a variable named sd there, it uses it immediately and stops searching. It never reaches the global environment or the stats package where the actual sd() function lives, effectively preventing a conflict.

Let’s create a wrapper function that can be used to get either or both metrics returned with a single function call. Do not rewrite your above two functions, call them inside the wrapper function (we would call the getRMSE() and getMAE() functions helper functions). When returning your values, give them appropriate names. The function should check that two numeric (atomic) vectors have been passed (consider is.vector(), is.atomic(), and is.numeric()). If not, a message should print and the function should exit. The function should return both metrics by default and include names. The behavior should be able to be changed using a character string of metrics to find.

get_metrics <- function(resp, pred, metrics = c("RMSE", "MAE"), ...) {
  
  # Check that both inputs are numeric atomic vectors
  if (!(is.vector(resp) && is.atomic(resp) && is.numeric(resp)) ||
      !(is.vector(pred) && is.atomic(pred) && is.numeric(pred))) {
    stop("Error: Both 'resp' and 'pred' must be numeric atomic vectors.")
  }
  
  # Initialize empty list to store named outputs
  results <- list()
  
  # Call helper functions based on character string provided
  if ("RMSE" %in% metrics) {
    results[["RMSE"]] <- getRMSE(resp, pred, ...)
  }
  
  if ("MAE" %in% metrics) {
    results[["MAE"]] <- getMAE(resp, pred, ...)
  }
  
  return(results)
}
